# Use Docker as the canonical runtime

Version 1 will define and test one canonical CPU-capable Docker image containing the specialized recognition runtime, PDF/UA authoring engine, fonts, rendering utilities, and validator. Users invoke a thin host-side `accessibilizer` launcher that transparently runs the pinned image, maps source and bundle paths, preserves interactive terminal behavior, and reaches a configured external OpenAI-compatible service; long `docker run` commands are not the normal UX. Conversion is officially supported on macOS and Linux hosts, including Apple Silicon and x86-64 where pinned model artifacts permit; optional NVIDIA acceleration may be added on Linux, while Windows is initially a PDF compatibility-test platform rather than a conversion host.
