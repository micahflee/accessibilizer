# Author the Semantic Layer as a full-size text overlay

ADR 0026 rejected the one-point-wide, zero-opacity `ActualText` overlay because macOS Preview derives accessibility text from the glyphs physically laid out on the page — so the narrow boxes produced single-letter fragments (`w`, `c`, …) instead of the complete strings, and the empty zero-opacity figure `Div` was dropped entirely. We replace that technique by drawing the real strings at a readable size across the page with text rendering mode 3, which produces no marks on screen or in the print path: the glyphs Preview reads now spell the complete heading, paragraph, Formula, and Figure strings. Each node occupies its own vertical band, top to bottom in Logical Reading Order, so no run overlaps another and the intended sequence is preserved. Because ADR 0026 showed Preview reads laid-out glyphs rather than the `ActualText`/`Alt` attributes for text, every string a reader must reach is drawn as a glyph run: the Formula draws its normalized math. A Figure is the exception — Preview treats it as an image and reads its `/Alt`, ignoring that element's own glyphs and `ActualText` — so the Figure carries the short Figure Alternative in `/Alt` (with matching glyphs so it is not dropped) while a complex Informative Figure additionally authors its Detailed Figure Description as a sibling `Caption` whose glyphs Preview reads like any other text element; a simple Informative Figure has no Detailed Figure Description and therefore no `Caption`. A Semantic Table is authored as a real PDF/UA `Table` rather than a single band: its optional `Caption`, its `TR` rows, and its `TH`/`TD` cells are tagged structure, each cell's text is drawn as an invisible mode-3 glyph run with the text on `ActualText`, a header cell's `Scope` associates the cells it labels, and a merged cell keeps its `RowSpan`/`ColSpan`; the cells carry no borders so the table adds no marks to the Visual Layer. `ActualText` and `Alt` remain on every structure element, and neither the `Caption` nor the table's rows and cells are treated as separate top-level semantic nodes, so the internal extraction, veraPDF PDF/UA-1, and visual-preservation gates pass unchanged. This spike prepares the artifact for the recorded human Preview and VoiceOver session tracked by #3; it does not by itself close #3.
